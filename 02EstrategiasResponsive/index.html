Graceful Degradation

Progressive Enhancement

La degradación elegante consiste en la construcción de la página web, de tal forma que los usuario con los navegadores más modernos tengan un cierto nivel de experiencia, pero tambien permite su uso a los usuarios con navegadores antiguos ofreciendo un nivel más bajo en su experiencia de usuario. Este nivel más bajo no es tan agradable de usar para los visitantes del sitio, pero aún así la página ofrece la funcionalidad mínima y suficiente para utilizar el sitio web. La estructura no se rompe, las cosas están accesibles y además también funcionan para ellos.

La mejora progresiva comienza con una base usable para todos y se incrementa la riqueza de la experiencia para quienes utilicen mejores y más modernos medios de acceso. Se empieza haciendo un nivel básico que todos los navegadores serán capaces de proporcionar cuando muestren la página web. Pero a partir de él se realizan mejoras que estarán disponibles de forma automática en los navegadores que puedan utilizarlas. Hoy o dentro de un año.

Conclusión

La degradación elegante comienza en una complejidad final, y trata de arreglar las cosas para las experiencias de usuario más bajas frente a la mejora progresiva que comienza en un mínimo que funciona de forma muy básica, pero que permite extenderla de forma constante en futuros entornos.

La degradación elegante significa pensar atrás y ser rehén del pasado, mientras que la mejora progresiva significa mirar al futuro siendo consciente del estado de la cuestión en el presente.

Adaptive Design

Es la capacidad que tiene un sitio web para ser visualizado correctamente independientemente del dispositivo que lo consuma.

No se trata exclusivamente de reacomodar la información sino de presentar el contenido de manera óptima para el dispositivo que el usuario este usando.

Responsive Web Design + Server Side

RESS es una combinación de RWD y AWD. Su objetivo es combinar las ventajas de ambos, responsivo y adaptativo, al mismo tiempo mediante el uso de la lógica del lado del servidor, para servir contenido diferente basado dinámicamente en las tecnologías del agente de usuario y el lado del cliente.

La idea tras RESS, es que podamos combinar las técnicas de diseño web responsivo y las técnicas del lado del servidor para traer una experiencia óptima a cada dispositivo. Esto significa que se atenderán solicitudes ligeramente diferentes a algunos dispositivos para una URL dada, pero aún usaremos técnicas de responsividad para todo lo que se muestre en dichos dispositivos.

RESS se ocupa de detectar 2 características con tecnologías del servidor y así servir el contenido adecuado para cada dispositivo:

  1. Versión del navegador
  2. Características del dispositivo

Las técnicas implementadas en el servidor podrán realizarse en el lenguaje que se esté desarrollando, no hay limitante de trabajar con uno particularmente.

Responsible Responsive Web Design

Es agregarle buenas prácticas de JavaScript a las buenas prácticas de CSS del Responsive Web Design

Responsive = Buenas prácticas de CSS

Responsible = Buenas prácticas de CSS y JS

Sus principios se basan en combinar el Responsive Design con prácticas responsables basadas en:

  • Usabilidad
  • Accesibilidad
  • Sostenibilidad
  • Rendimiento
