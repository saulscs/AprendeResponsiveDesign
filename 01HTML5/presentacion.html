UNIDADES DE MEDIDA

  • Absolutas

    px, pt, pc, cm, mm, in

  • Relativas

    • al Contenedor
      • % (Porcentajes)
    • a la Tipografía
      • em basada en la anchura de la letra
      • ex basada en la altura de la letra
      • rem es el em del elemento raiz (html)
    • al Viewport
      • vw anchura del viewport
      • vh altura del viewport
      • vmax entre vw y vh toma el que tenga mayor valor
      • vmin entre vw y vh toma el que tenga menor valor

  • Equivalencias

    100% = 1em = 16px = 12pt

PATRONES

  • Son buenas prácticas de como hacer CSS basados en DRY (Don't Repeat Yourself), útiles cuando trabajamos en equípo
  • Código CSS reusable, modular y óptimo

  1. FRAMEWORKS

    • VENTAJAS

      • Facil de probar
      • Flujo de trabajo definido
      • Sintaxis similar
      • Fuentes de Aprendizaje
      • Ideal para trabajos que eran para ayer

    • DESVENTAJAS

      • Convenciones ajenas
      • Globales
      • Impersonalizables
      • Incompatibilidades
      • Sobrecarga del DOM
      • Generan codependencia
      • Mala idea para aprender a maquetar

    • EJEMPLOS

      • Bootstrap
      • Foundation
      • Pure CSS
      • jQueryMobile

  2. OOCSS

    CSS Orientado a Objetos

  3. BEM

    Bloque-Elemento-Modificador

    Estructura

    • .bloque { ... }
    • .bloque__elemento { ... }
    • .bloque--modificador { ... }

    Ejemplo

    • .menu { ... }
    • .menu__item { ... }
    • .menu__item--hover{ ... }

  4. CCSS

    Componentes CSS

    Estructura

    • ComponentName Standard Components (B)
    • ComponentName-elementName Component's Element (E)
    • ComponentName--modifierName Component's Modifier (M)

    Ejemplo

    • u-className Global base/utility classes
    • img-className Global image classes
    • animate-className Global animation classes

BUENAS PRÁCTICAS DE CÓDIGO CSS

  • Separar nombre de selectores con el guión medio (-) y escribir en minúsculas
  • Preferir links sobre @import para llamar las hojas de estilo
  • Definir siempre un font-size al elemento root (html) y hacerlo en px
  • Ser cuidadoso con la definición de reglas a selectores de etiquetas
  • Evitar definición de selectores de ID
  • Usar en su mayoria selectores de clases
  • Aprovechar los pseudo-selectores y unidades de medida disponibles
  • Tener precaución con los shorthand de CSS
    • padding, margin, font, background, border, border-radius
    • preferible background-color: #FFF que background: #FFF
  • Normalizar los estilos iniciales de nuestras etiquetas HTML
    • Reset.css
    • Normalize.css
    • Definir uno propio que se adapte a nuestra técnica de maquetación y no te olvides del box-sizing
  • Maquetar bajo un enfoque Mobile First

ORDENAR CÓDIGO CSS

  1. Ir de lo General a lo Particular (Aprovechar la Cascada CSS)
  2. Código para resetear
  3. Selectores de Etiqueta - (Átomos)
  4. Selectores de ID si hubiera - (Átomos)
  5. Clases de Estilo: Tipografías, Colores, Tamaños, Estados, etc - (Moléculas)
  6. Clases de Estructura: Visualización y Maquetación - (Moléculas)
  7. Clases de Objetos específicos: Botones, enlaces, textos, items, imágenes, celdas etc - (Organismos)
  8. Clases de Componentes específicos: Logos, listas, formularios, tablas, figuras, acordeónes, menús, páneles, widgets - (Organismos)
  9. Clases de Secciones específicas: Cabecerá, piés de páginas, navegaciones, contenidos principales, secundarios, galerías, etc - (Organismos)
  10. Clases de Contenido específico de páginas: Página de Autenticación de Usuarios, Galería Multimedia del Home, Formulario de la zona de Contacto, Página de Productos, etc - (Plantillas y Páginas)
  11. Mediaqueries
    • Preferir Mobile First a Desktop First
    • Seguir el patrón del Diseño Atómico
